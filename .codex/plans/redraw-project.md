# AI 短剧“转绘”项目开发计划

## 摘要

- 计划文档目标路径：`.codex/plans/redraw-project.md`；实施阶段第一步将本定稿完整写入该文件。
- 在“我的项目 → 新建项目”的“基于小说原文 / 基于剧本 / 基于分镜表”中增加第四类“转绘”，正式使用 `projectType: "redraw"`。
- 每个转绘项目首版绑定一个源视频/单集，支持 MP4、MOV、WebM，最大 20 分钟、2GB。
- “一比一复刻”定义为：剧情事件、对白、人物动作、镜头顺序、镜头起止时间和原音轨不变；人物、服装、场景、道具及画面媒介按目标风格转换，不承诺逐像素一致。
- 采用分阶段、可确认、可续跑工作流，所有业务状态落 SQLite 和 OSS，不依赖浏览器 `localStorage`。

## 一、项目入口与转绘工作区

- 扩展 `ProjectType`、标签、别名和项目增删改校验，加入 `redraw`；同步更新所有“仅支持三种项目类型”的错误信息。
- 为内置前端增加 `redraw-studio.js/.css`，注册 `/#/redraw?projectId=<id>`，复用现有工作台布局。
- 由于完整 Vue 源码不在仓库，通过带签名校验的确定性补丁修改 `data/web/index.html` 中三处固定逻辑：
  - 新建/编辑项目类型增加“转绘”。
  - 项目卡片正确显示“转绘”标签。
  - 点击转绘项目进入独立转绘工作区。
- 工作区依次展示：
  1. 源视频与目标风格。
  2. 视频分析与转绘剧本。
  3. 原始资产、原始资产图。
  4. 衍生资产、衍生资产图。
  5. 分镜表、分镜面板、分镜图。
  6. 视频片段、保真复核、最终成片。
- 提供“执行当前步骤”“完成本阶段”“仅重试失败项”和局部强制重生成。
- 在分析确认、资产确认、分镜确认和视频生成费用确认处设置人工检查点，不允许一键流程跳过。
- 上游数据改变后自动使下游结果变为“已过期”：
  - 修改目标风格：资产及后续步骤过期。
  - 修改镜头边界、动作或对白：剧本及后续步骤过期。
  - 修改资产图：关联分镜图、视频片段和成片过期。

## 二、Agent 简易配置与高级配置

### 简易配置

- 新增主配置项：

  `redrawAgent`

- 中文名称为“转绘Agent”，说明为“用于源视频理解、转绘剧本、资产映射、分镜生产和保真复核”。
- 简易模式下，所有 `redrawAgent:*` 子任务统一解析到 `redrawAgent` 配置的模型，沿用现有 `resolveModelName()` 的主键回退机制。
- 简易模式所选模型必须支持：
  - 文本输入。
  - 图片输入。
  - 视频输入。
  - 工具调用。
- 未满足能力要求时禁止开始视频分析，并明确列出缺失能力，不静默改用普通文本模型。

### 高级配置

新增以下独立配置项：

| 配置键 | 名称 | 职责 | 最低模型能力 |
|---|---|---|---|
| `redrawAgent:decisionAgent` | 转绘Agent：决策层 | 检查前置条件、选择允许的工具和当前工作步骤 | 文本、工具调用 |
| `redrawAgent:supervisionAgent` | 转绘Agent：监督层 | 检查分析、剧本、资产和分镜中间结果是否违背一比一规则 | 文本、图片、工具调用 |
| `redrawAgent:videoAnalysisAgent` | 转绘Agent：视频分析 | 分析镜头边界、人物动作、对白、场景、运镜、音效和源风格 | 视频、图片、工具调用 |
| `redrawAgent:scriptAgent` | 转绘Agent：剧本生成 | 将结构化镜头分析写成不增删剧情的转绘剧本 | 文本、工具调用 |
| `redrawAgent:assetMappingAgent` | 转绘Agent：资产映射 | 建立源人物/场景/道具与目标风格资产、衍生状态的映射 | 文本、图片、工具调用 |
| `redrawAgent:storyboardAgent` | 转绘Agent：分镜生成 | 生成分镜表、分镜面板内容和分镜图提示词 | 文本、图片、工具调用 |
| `redrawAgent:fidelitySupervisorAgent` | 转绘Agent：保真监督 | 比较源片段和生成片段，评分并产生重试修正意见 | 视频、工具调用 |

- 决策层只能调用后端允许的工具，不能绕过确定性工作流状态机、人工确认点或模型能力校验。
- 普通监督层负责剧本、资产、分镜等中间数据；保真监督层专门负责源视频与生成视频的逐镜比较，二者职责不合并。
- 高级模式允许不同供应商承担不同任务，但系统不会自动替用户切换供应商或产生未确认的费用。

### 配置初始化与兼容

- 在 `initDB` 中为新数据库加入主配置和全部高级配置。
- 在 `fixDB` 中按 `key` 幂等补充缺失项，已有数据库升级时不重复插入、不覆盖用户已有模型选择。
- 扩展 `AiType`、可用键列表、模型解析与任务记录，使所有转绘子 Agent 可由现有 AI 工具统一调用。
- 现有 Agent 配置接口会将：
  - 不含冒号的 `redrawAgent` 放入简易配置。
  - 含冒号的子 Agent 放入高级配置。
- 后端在保存配置和运行任务时再次校验能力，防止通过旧前端或直接 API 绑定不兼容模型。
- 图片生成模型和视频生成模型仍属于项目配置：
  - Agent 模型负责理解、规划、结构化数据和复核。
  - 图片模型负责资产图、衍生资产图和分镜图。
  - 视频模型负责最终转绘片段生成。

## 三、视频处理与多模态分析

- 引入固定版本的 FFmpeg/FFprobe 运行时，配置 Electron 解包、跨平台路径解析和许可证声明，不依赖系统 PATH。
- 源视频使用流式上传，不转 Base64；校验文件大小、容器、视频流、时长和安全路径，异常时删除临时文件。
- 分析开始前允许替换源视频；分析开始后必须执行明确的“重置转绘流程”，清除下游结果后才能更换。
- 首版只接受与项目设置匹配的 16:9 或 9:16 视频，比例误差不超过 2%；其他比例直接阻止分析。
- 本地媒体工具先生成：
  - 媒体元数据。
  - 镜头边界候选。
  - 低码率分析代理。
  - 镜头片段。
  - 关键帧。
  - 音频和字幕流信息。
- 视频分析 Agent 按片段调用受控工具：
  - `probe_source_video`：读取媒体、音轨和字幕信息。
  - `list_shot_candidates`：读取镜头边界候选。
  - `load_segment_evidence`：取得指定片段和关键帧。
  - `save_shot_analysis`：保存动作、对白、场景、运镜、音效和资产线索。
  - `validate_timeline`：检查镜头时间轴连续、无重叠、无缺口。
- 分析结果包含：
  - 源视频画面风格。
  - 带时间码的对白和字幕。
  - 角色、场景、道具清单。
  - 人物动作和情绪。
  - 景别、视角和运镜。
  - 音效和音乐信息。
  - 连续镜头时间轴。
- 用户可通过同步视频播放器逐镜修改边界、对白、动作和资产关联，确认后才进入剧本阶段。
- 目标风格由现有视觉手册、自由描述和参考图共同定义；人物、服装、场景、道具和媒介质感默认全部转换，并提供逐项开关。

## 四、剧本、资产和分镜生产

- 分析确认后创建一份现有 `o_script` 剧本，严格按源时间轴编排，不增加、删除、翻译或改写剧情与对白。
- “原始资产”表示目标风格下的角色、场景和道具标准设定。
- “衍生资产”表示同一资产在不同服装、状态、时间或场景中的目标风格变体；源视频画面证据单独保存。
- 原始资产图同时参考：
  - 源视频中的角色/场景/道具裁剪图。
  - 用户上传的目标人物或风格参考图。
  - 项目视觉手册和风格转换说明。
- 扩展图片批量接口支持 `referenceList[]`，保留原有单个 `base64` 参数兼容性；多参考时要求图片模型声明相应能力。
- 衍生资产根据结构化镜头时间轴生成并关联源镜头，继续使用 `o_assets.assetsId` 表示父子关系。
- 资产确认后，将源镜头物化为分镜表和分镜面板，包含：
  - 原始起止时间。
  - 画面和动作描述。
  - 景别、视角和运镜。
  - 对白、音效和情绪。
  - 原始资产与衍生资产关联。
- 正常镜头对应一个生成片段。
- 超过视频模型最大时长的镜头，在低运动区域拆成带重叠的内部片段，合成时融合；界面仍按一个源镜头分组。
- 短于模型最小时长的片段按模型最小时长生成，再按源时间轴精确裁切。
- 分镜图以源关键帧作为构图参考，同时引用目标风格原始资产图和衍生资产图，结果继续落入现有 `o_storyboard`。

## 五、视频生成、保真复核与成片

- 转绘项目的视频模型必须在 `mode` 中声明 `videoReference:N`；不兼容时阻止创建或最终生成。
- 每个生成片段携带对应源视频片段作为视频参考，并附带目标资产图、分镜图和严格约束提示词。
- 视频模型生成时关闭模型音频，避免覆盖原声。
- 扩展视频批量接口参考来源，新增 `sources: "redrawSegment"`，按视频 MIME 传递；其他项目继续使用原有资产/分镜逻辑。
- 保真监督模型逐片比较源片段和生成片段，检查：
  - 人物数量和身份对应。
  - 剧情动作和事件顺序。
  - 景别、构图和运镜。
  - 动作发生时间。
  - 是否出现新增或缺失剧情。
- 默认通过线为 85/100；缺失关键动作、增删剧情事件或镜头顺序错误属于硬失败。
- 未通过的片段自动携带监督反馈重试最多 2 次，即单片段最多生成 3 次。
- 两次重试后仍失败则标记“需要人工确认”；用户可继续重生成或手动接受，未全部通过/接受前禁止最终合成。
- 最终合成按源时间轴裁切、融合和拼接片段，统一到源分辨率和帧率，再挂载原视频音轨。
- 音轨可直接复用时 stream-copy，否则仅做容器兼容转码，不改变内容和时间轴。
- 分析阶段生成可编辑字幕；检测到嵌入字幕或硬字幕时默认重新烧录，同时导出 SRT，用户可关闭烧录。
- 输出单个 MP4，并保留成片历史、质量报告和 SRT。
- 验收标准：
  - 镜头边界误差不超过 1 个源视频帧。
  - 成片总时长误差不超过 1 个源视频帧。
  - 音画同步误差不超过 40ms。
  - 剧情、对白和镜头顺序无增删。

## 六、数据结构与接口

### 新增数据表

- `o_redrawSource`：源文件、媒体元数据、源/目标风格、分析状态和当前剧本。
- `o_redrawShot`：源镜头时间轴、对白、动作、镜头语言和确认状态。
- `o_redrawSegment`：内部生成片段及其分镜、轨道、视频尝试和保真结果映射。
- `o_redrawReference`：源资产证据、目标风格图及其资产关联。
- `o_redrawOutput`：最终视频、SRT、合成状态和验收指标。

### 工作流状态

- 扩展 `o_workflowStepRun` 的输入摘要和元数据，用于判断结果是否过期。
- 应用重启时将中断的上传和运行任务转为可重试失败状态。
- 新增 `RedrawStep`：
  - `analyzeSource`
  - `createScript`
  - `createOriginalAssets`
  - `generateOriginalAssetImages`
  - `createDerivedAssets`
  - `generateDerivedAssetImages`
  - `buildStoryboards`
  - `generateStoryboardImages`
  - `generateVideoPrompts`
  - `generateVideos`
  - `assembleOutput`

### 新增接口

- `/api/redraw/workspace/get|update`
- `/api/redraw/source/upload|reset`
- `/api/redraw/reference/upload|delete`
- `/api/redraw/shot/update|confirm`
- `/api/redraw/workflow/run|progress`
- `/api/redraw/segment/accept`

- 所有写接口校验 `projectId` 必须属于 `redraw` 项目。
- 运行中的同一步骤重复提交返回冲突，不创建重复资产、分镜或视频轨道。
- 删除转绘项目时同步清理专属表和媒体文件。
- 初始化和 `fixDB` 同时实现幂等迁移，已有项目无需人工迁移。

## 七、测试计划

- 测试 `redrawAgent` 在简易模式下正确承接全部子任务。
- 测试高级模式下每个子 Agent 使用各自模型，不错误回退到其他子配置。
- 测试已有数据库升级后补齐 Agent 配置但不覆盖已有选择、不产生重复记录。
- 测试不兼容模型在配置保存和任务执行两个阶段都被拒绝。
- 单元测试项目类型规范化、上传限制、路径逃逸、媒体比例、镜头连续性、长短片段规划、字幕时间码和输入摘要失效。
- 使用临时合成视频验证 FFmpeg 探测、抽帧、切片、重叠融合、原声挂载、SRT 输出及最终时长。
- 使用伪造多模态、图片和视频供应商跑完整流程，验证工具调用、对白不被改写、幂等重试、失败恢复和最多两次自动重试。
- 验证“我的项目”弹窗只出现一个“转绘”选项，卡片标签、编辑、删除、打开路由和前进/后退正确。
- 回归执行：
  - `yarn lint`
  - 现有分镜导入测试
  - 新增 `yarn test:redraw`
  - 转绘前端脚本 `node --check`
  - `yarn build`
  - `git diff --check`
- 执行 Electron `yarn pack` 冒烟测试，确认 FFmpeg/FFprobe 和转绘 Agent 配置在开发、生产及 Electron 环境均可用。

## 假设与默认值

- 单项目单视频、20 分钟/2GB 上限，不包含多集共享资产、超长视频和批量导入。
- 原音轨、原语言、角色名称和对白不翻译、不重新配音；无音轨的视频保持静音。
- 项目比例必须与源视频一致，首版不支持 1:1、4:3 等其他比例。
- 自动保真重试可能使单片段最多产生 3 次视频生成费用，提交前显示片段数量和最大尝试次数。
- 用户拥有源视频和目标参考图的合法转绘与使用权。
- 实施过程只使用当前唯一仓库和 `master` 分支，不创建其他分支、仓库或 worktree。
